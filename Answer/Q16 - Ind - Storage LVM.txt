16. Storage LVM
Create & use a logicalVolume
Create a playbook called /home/admin/ansible/ansible/lv.yml that runs on all managed nodes that does the following:
* Creates a logical volume with these requirement:
- The logical Volume is Created in the research volume group
- The logical volume name is data
- The logical volume size is 1500 Mib
* Format the logical volume with the ext4 file system
* if the requested logical volume size cannot be created, the error message Could not create
logical volume of that size should be displayed and size 800 MiB should be used instead.
* If the volume research does not exist, the error message volume group does not exist should be
displayed
* Does NOT mount the logical volume in any way.
====
ansible/lv.yml
---
- name: Storage LVM - Create and use a Logical Volume
  hosts: all
  tasks:

    - name: Check if volume group 'research' exists
      command: vgs research
      register: vg_status
      ignore_errors: yes
      changed_when: false

    - name: Display error if volume group does not exist
      ansible.builtin.debug:
        msg: "volume group does not exist"
      when: vg_status is failed

    - block:  # ← bloque exterior: solo corre si VG existe

        - block:  # ← bloque interior: intenta 1500 MiB
            - name: Create logical volume 'data' with 1500 MiB
              community.general.lvol:
                vg: research
                lv: data
                size: 1500m

          rescue: # ← solo corre si falla el bloque interior
            - name: Display error - could not create LV of that size
              ansible.builtin.debug:
                msg: "Could not create logical volume of that size"

            - name: Create logical volume 'data' with 800 MiB (fallback)
              community.general.lvol:
                vg: research
                lv: data
                size: 800m

        - name: Format logical volume with ext4  # ← dentro del bloque exterior
          ansible.builtin.filesystem:
            fstype: ext4
            dev: /dev/research/data

      when: vg_status is not failed
=====
ansible node5 -m shell -a "lvremove -y /dev/research/data"
sudo lvremove /dev/research/data

** Preparar lab
---
- name: Limpiar y preparar VG research en /dev/sdb
  hosts: node5 node4
  become: true
  tasks:

    # ── LIMPIEZA ────────────────────────────────────────────────────────────

    - name: Obtener LVs asociados a /dev/sdb
      shell: lvs --noheadings -o lv_path,devices | grep '/dev/sdb' | awk '{print $1}'
      register: lvs_en_sdb
      changed_when: false
      failed_when: false

    - name: Desactivar y eliminar LVs asociados a /dev/sdb
      command: lvremove -f {{ item }}
      loop: "{{ lvs_en_sdb.stdout_lines }}"
      when: lvs_en_sdb.stdout_lines | length > 0

    - name: Obtener VG asociado a /dev/sdb
      shell: pvs --noheadings -o vg_name /dev/sdb 2>/dev/null | tr -d ' '
      register: vg_en_sdb
      changed_when: false
      failed_when: false

    - name: Eliminar VG asociado a /dev/sdb
      command: vgremove -f {{ vg_en_sdb.stdout }}
      when:
        - vg_en_sdb.stdout is defined
        - vg_en_sdb.stdout | length > 0

    - name: Eliminar PV en /dev/sdb
      command: pvremove -ff -y /dev/sdb
      failed_when: false

    - name: Limpiar firmas del disco con wipefs
      command: wipefs -a /dev/sdb

    # ── PREPARACIÓN ─────────────────────────────────────────────────────────

    - name: Crear PV en /dev/sdb
      command: pvcreate /dev/sdb

    - name: Crear VG research
      command: vgcreate research /dev/sdb

    # ── VERIFICACIÓN ────────────────────────────────────────────────────────

    - name: Mostrar estado de PVs
      command: pvs
      register: pvs_result
      changed_when: false

    - name: Mostrar estado de VGs
      command: vgs
      register: vgs_result
      changed_when: false

    - name: Imprimir PVs
      debug:
        var: pvs_result.stdout

    - name: Imprimir VGs
      debug:
        var: vgs_result.stdout

🔎 Pasos para crear discos en Oracle VM VirtualBox
Apaga el nodo (ejemplo: node1).
En VirtualBox, selecciona la VM → Apagar.
Agrega un disco nuevo
Configuración → Almacenamiento → Controlador SATA → Agregar disco duro.
Crea un disco virtual nuevo (VDI) de, por ejemplo, 2 GB.
Asegúrate de que esté marcado como disco adicional, no el de arranque.
Inicia la VM
Arranca el nodo y verifica el nuevo disco:
lsblk
Verás algo como /dev/sdb (además de tu disco principal /dev/sda).

- Crea el Volume Group
En el nodo:
sudo pvcreate /dev/sdb
sudo vgcreate research /dev/sdb
Ahora tienes el grupo research listo para que tu playbook cree el LV.

📂 Ejemplo de flujo para todos los nodos
Si quieres automatizar la creación de discos en varios nodos:
En VirtualBox, agrega un disco extra a cada VM (ej. /dev/sdb).
Luego usa Ansible para inicializar el VG en todos los nodos:

yaml
---
- name: Preparar VG research en todos los nodos
  hosts: node5
  become: true
  tasks:
    - name: Crear PV en /dev/sdb
      command: pvcreate /dev/sdb
      args:
        creates: /dev/sdb

    - name: Crear VG research
      command: vgcreate research /dev/sdb
      args:
        creates: /dev/research

    - name: Mostrar resultado de vgs
      command: vgs
      register: vgs_result

    - name: Imprimir salida de vgs
      debug:
        var: vgs_result.stdout
