8. Use roles from Ansible Galaxy Create a playbook called /home/admin/ansible/roles.yml
* The playbook contains a play that runs on host in the balances host group and uses the balancer role.
- This role configures a service to load balance web server request between hosts in the webserver host group.
- Browsing to host in the balances host group ( for example
http://system5.domain1.example.com) produces the following output: Welcome to
system3.domain1.example.com on 172.24.1.8
- Reloading the Browser produces output from the AlterNet web server:
Welcome to system4.domain1.example.com on 172.24.1.9
* The Playbook contains a play the runs on hosts in webserver host group and user the phpinfo role. 
- Browsing to host in the webserver host group with the URL /hello.php produces the following output : Hello PHP World from FQDN
- For example Browsing to http://system3.domain1.example.com/hello.php produces the
following output: Hello PHP World from system3.domain1.example.com along with various
details of the PHP configuration include the version of PHP that is installed.
@powered by JSNH
- Similarly, Browsing to http://system4.doamin1.example.com/hello.php, produces the following output: Hello PHP World from system4.domain1.example.com along with various details of the PHP configuration including the version of PHP that is installed.

ansible/roles/phpinfo/tasks/main.yml (Instala php, arranca servicios, abre puerto 80 firewall, copia plantilla hello.j2
---
- name: Instalar Apache y PHP
  yum:
    name:
      - httpd
      - php
    state: present

- name: Habilitar y arrancar Apache
  service:
    name: httpd
    state: started
    enabled: yes

- name: allow httpd through firewall
  ansible.builtin.firewalld:
    service: http
    permanent: yes
    immediate: yes
    state: enabled

- name: Copiar archivo hello.php
  template:
    src: hello.j2
    dest: /var/www/html/hello.php
=====
* Funcionamiento
El playbook principal (roles.yml) orquesta todo: aplica balancer en el grupo balancers y phpinfo en el grupo webservers.

El rol balancer configura HAProxy para que el host balancer (192.168.0.115) distribuya las peticiones HTTP entre los webservers (192.168.0.113 y 192.168.0.114).

El rol phpinfo prepara cada webserver con Apache + PHP y genera la página hello.php que muestra el mensaje dinámico con el FQDN y la salida de phpinfo().

Al acceder a http://192.168.0.115/hello.php, HAProxy alterna las respuestas entre los dos webservers, mostrando el mensaje correcto de cada uno.

roles/balancer/tasks/main.yml
---
- name: Instalar haproxy
  ansible.builtin.yum:
    name: haproxy
    state: present
- name: Copiar configuración haproxy
  ansible.builtin.template:
    src: haproxy.cfg.j2
    dest: /etc/haproxy/haproxy.cfg
  notify: restart haproxy
- name: Habilitar y arrancar haproxy
  ansible.builtin.service:
    name: haproxy
    state: started
    enabled: true

++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++
++++++++ Preparación de role balancer ++++++++
Configuración copiada roles/balancer/templates/haproxy.cfg.j2
global
    log /dev/log    local0
    log /dev/log    local1 notice
    daemon
    maxconn 256

defaults
    log     global
    mode    http
    option  httplog
    option  dontlognull
    retries 3
    timeout connect 5000ms
    timeout client  50000ms
    timeout server  50000ms

frontend http_front
    bind *:80
    default_backend webservers

backend webservers
    balance roundrobin
    server node3 192.168.0.113:80 check
    server node4 192.168.0.114:80 check
====
Tareas configuradas roles/balancer/tasks/main.yml
- name: Instalar HAProxy
  yum:
    name: haproxy
    state: present

- name: Copiar configuración de HAProxy
  template:
    src: haproxy.cfg.j2
    dest: /etc/haproxy/haproxy.cfg
    owner: root
    group: root
    mode: '0644'

- name: Habilitar y arrancar HAProxy
  service:
    name: haproxy
    state: started
    enabled: yes

- name: Permitir tráfico HTTP en firewall
  ansible.posix.firewalld:
    service: http
    permanent: yes
    state: enabled

- name: Recargar firewall
  ansible.builtin.command: firewall-cmd --reload
