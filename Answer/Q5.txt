5. Install roles using Ansible Galaxy
Download and install role to /home/admin/ansible/roles from following URL:
playbook name shoud be requirement.yml
http://server.network.example.com/materials/haproxy.tar
- The Name of the role should be balancer
http://server.network.example.com/materials/phpinfo.tar
- The Name of the role should be phpinfo
====
[admin@control ~]$ cd ansible/roles/
[admin@control roles]$ vim requirmental.yml

- src: https://github.com/geerlingguy/ansible-role-haproxy.git
  name: balancer

- src: https://github.com/bbatsche/Ansible-PHP-Site-Role.git
  name: phpinfo 

:wq!

[admin@control roles]$ ansible-galaxy install -r requirmental.yml

Note: Link is given in Exam , I am using this link for pratical purpose.

====
Option 1 - Install collections
ansible-galaxy collection install -p collections/ URL
for each collection
====
Option 2
[admin@control ~]$ cd ansible/roles/
[admin@control roles]$ vim requirment.yml
- src: https://github.com/geerlingguy/ansible-role-haproxy.git
  name: balancer
- src: https://github.com/bbatsche/Ansible-PHP-Site-Role.git
  name: phpinfo 
[admin@control roles]$ ansible-galaxy install -r requirement.yml
or
ansible-galaxy collection install -r collections/requirements.yml -p collections
===
Prueba, verificación
ansible-galaxy list
====
Inicializa roles vacíos:
/home/admin/ansible/roles
ansible-galaxy init balancer
ansible-galaxy init phpinfo
====
Edita tasks/main.yml en cada rol para simular la lógica:
* En balancer/tasks/main.yml podrías poner:
yaml
- name: Install HAProxy
  package:
    name: haproxy
    state: present
* En phpinfo/tasks/main.yml podrías poner:
yaml
- name: Deploy phpinfo page
  copy:
    dest: /var/www/html/phpinfo.php
    content: "<?php phpinfo(); ?>"
